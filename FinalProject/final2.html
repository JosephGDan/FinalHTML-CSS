There are many different options about what should be in the top when considering HTML and CSS coding. After much research and deliberation, I have come up with the top 10 that all articles had in common.

Use Proper Document Structure

Make sure you are including the right document type and elements inside of your coding. Things like < html >, < head >, < body >, < main >, etc. Since web pages do not always render correctly, it is important to be consistent when using the right document structure.

Don't Use Inline Styles

Never mix an internal style sheet with an external style sheet. Always learn to use the external style sheet when possible and creating presentations. If you need another style, just code another sheet and call the one that you would like to use. Make sure to place your external style sheet within the < head > tag.

Image Attribution

Although it is not required to use an alt attribute with your images, programmers should get into the habit of using descriptive text attributes on their images whenever possible (because it provides context to screen readers) but leave the value blank if the image has no important relevance to anything. This will help with validation and give greater accessibility.

Validate Frequently

Using the W3C Standards will help your web pages load faster. Validate often to make sure your coding has no errors and never wait for the end of the project because that is where the most errors will occur. It will save you time, especially if your code is a lengthy one. The links below will provide the validation for both HTML and CSS.

  • W3C Validator ( HTML )
  • CSS Validator

Use Meaningful Tags

Making sure that your title tags are search-engine friendly and be sure to use descriptivemeta tags that define the purpose of your web page. This will ensure that people find your page quicker.

Use DIV Tags with Class/ID Selectors Wisely

Using div tag for major sections is fine but do not get into the habit of always wanting to use them. Know the difference between Class and ID selectors. Apply classes to multiple elements that have the same style properties but use Ids for the unique style elements of a single type. Class selectors will always have a period before the name (ex. .features) while the ID selectors have # before the name (ex. #features). Remember that HTML markup will not care if your tags are uppercase or not but stay away from this and make your tags lowercase.

Using Good Semantics

Making a navigation list should always be an unordered < ul > or ordered < ol > list. Using a paragraph < p > instead of breaks < br > is also considered good practice. Knowing when to use the right tags in the right place is important when organizing your markup to ensure that it will look clean.

Commenting Your Coding

Commenting your coding is not really necessary considering the fact that you will always have to return to your markup on a different day and thus, could only confuse yourself and others but knowing when to comment out your coding can be important. For example, commenting out a closing div tag with which one it started with could be helpful when the user has multiple div tags.

Use a CSS Reset File

Sometimes a file will get an extra bit of padding or something along those lines. This happens because every browser has its own default stylesheet that it uses to make unstyled websites appear more legible. Creating this reset file will give a consistent baseline for your HTML file to follow and prevent this from happening.

Put JavaScript Files at the Bottom

By doing this, all your HTMl content will load first and then the Javascripting will load second. This ensures that the content is loaded properly.
